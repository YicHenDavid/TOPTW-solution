1. 问题定义 — TOPTW

TOPTW (Team Orienteering Problem with Time Windows)：m 个队员，n 个地点（含起终点）， 每个地点 i 有分数 s_i、服务时间 d_i、时间窗 [o_i, c_i]。 每人时间预算 T_{max}。目标：最大化总收集分数，每个地点最多访问一次。

字段C++ 类型含义
nint总节点数（含 depot_start=0, depot_end=n-1）
mint队员数量
Tmaxdouble每队时间预算
x, yvector<double>节点坐标（[0,100] 区间）
distvector<vector<double>>距离矩阵（欧几里得）
scoresvector<int>各节点分数。起点/终点 = 0
servicevector<double>服务时间。起点/终点 = 0
tw_open, tw_closevector<double>时间窗 [o, c]。起点/终点 = [0, Tmax]

2. 解的表示 & 解码器

解 = m 条路径的集合，每条路径是 vector<int> 节点序列，首尾固定为起点和终点。
采用 间接表达 (encode–decode)：只存访问顺序，到达时间由解码器递推计算。

解码器 — evaluate_route()

遍历路径，递推每个节点的到达时间，每一步做两项检查：

arrive_k = \max(arrive_{k-1} + d_i + dist(i,j),\; o_j)
变量含义
arriveₖ在第 k 个位置（节点 j）的实际到达时间
arriveₖ₋₁ + dᵢ + dist(i,j)自然到达时间 = 上一节点到达 + 服务 + 旅行
oⱼ节点 j 的时间窗开门时间。早到则等待

两条硬性检查（任意一条失败 → 路径不可行）：

  1. 时间窗违规：arrive_k > c_j（到达时窗口已关闭）
  2. 回不到终点：arrive_k + d_j + dist(j, end) > T_{max}

返回值：std::optional<vector<double>> — 可行则含到达时间数组，不可行则为 std::nullopt

3. 初始解构造 — GRASP

GRASP (Greedy Randomized Adaptive Search Procedure)：贪心插入 + RCL (Restricted Candidate List) 随机化。
每次从「空路径」开始（每队只有 [起点, 终点]），反复将未访问节点插入到性价比最高的位置。

性价比：

ratio(node, team, pos) = \frac{scores[node]}{\max(insertion\_cost,\; 0.01)}

其中 insertion_cost = 插入该节点后终点到达时间的增量。

RCL 随机化：

  • \alpha = 0：纯贪心，选性价比最高的
  • \alpha > 0：候选按性价比降序排列，从前 \alpha \times |candidates| 个中随机选一个

多起点策略：10 次构造，\alpha 从 0 递增到 0.27，逐步增加随机性 → 取最优。

4. 局部搜索算子

三个算子按顺序迭代执行，直到总分不再提升（first-improvement 策略）。

4.1 2-opt（路径内反转）

操作：选 i < j，断开边 \{i,i+1\} 和 \{j,j+1\}，重连为 \{i,j\} 和 \{i+1,j+1\}。中间段被反转。

目的：消除路径交叉，缩短总距离。枚举所有 (i,j) 对，保留使终点到达时间最短的解。

复杂度：O(L^2)，L = 路径长度。

4.2 Relocate（路径间迁移）

操作：从路线 A 移除节点 v，插入到路线 B 的某个位置。

目的：释放 A 的时间 + B 利用空闲收编。枚举所有 (节点, 源路线, 目标路线, 插入位置)。

复杂度：O(m^2 \cdot n \cdot L^2)。

4.3 Exchange（路径间交换）

操作：路线 A 的节点 a ⇄ 路线 B 的节点 b 互换。

目的：两个节点各得其所。枚举所有 (路线对, 节点对)。

复杂度：O(m^2 \cdot L^2)。

5. ALNS 框架

🧠 ALNS 的定位
ALNS 是全局优化框架——它不亲自做构造或局部搜索，而是调度 destroy/repair 算子
来探索搜索空间，并通过自适应权重自动学习"哪个算子组合在当前阶段最有效"。

核心循环：

1. 从 GRASP + LS 得到初始解 x
2. repeat:
3.    轮盘赌选 destroy d + repair r（根据历史权重）
4.    x' ← r(d(x))                  // destroy → repair
5.    对 x' 做局部搜索 (2-opt + relocate + exchange)
6.    SA 准则决定是否接受 x'
7.    根据 x' 质量更新 d 和 r 的权重
8.    降温 T ← α × T
9. until 终止

5.1 Destroy 算子（3 个）

算子策略移除什么节点
Random 随机选择 从已访问节点中随机选 q 个（q \approx 15\% 的已访问节点）
Related 按距离聚类 随机选一个种子节点，移除离种子最近的 q 个节点（地理相近的节点更容易通过重组改善）
Worst 按性价比 移除 score / insertion_cost 比值最低的 q 个节点（"性价比最差的"，移除后可能换来更好的节点）

5.2 Repair 算子（2 个）

算子策略如何重插
Greedy 贪心最佳优先 每轮选 score/cost 最大 的 (节点, 路线, 位置) 组合插入
Regret-2 后悔值优先 每轮选 regret = 第 2 优 cost − 最优 cost 最大的节点先插入
直觉：不插这个节点会"最后悔"，因为它只有一个好的插入位置

5.3 自适应权重机制

轮盘赌选择：每轮根据当前权重，按概率选择一个 destroy 和一个 repair：

p_j^- = \frac{\rho_j^-}{\sum_{k \in \Omega^-} \rho_k^-}, \qquad p_j^+ = \frac{\rho_j^+}{\sum_{k \in \Omega^+} \rho_k^+}
变量含义
\Omega^-, \Omega^+destroy / repair 算子集合
\rho_j算子 j 的当前权重（越大越容易被选中）
p_j算子 j 被选中的概率 = 自身权重 / 总权重

权重更新（只更新本轮被选中的算子）：

\rho_a^- = \lambda \rho_a^- + (1-\lambda)\psi, \qquad \rho_b^+ = \lambda \rho_b^+ + (1-\lambda)\psi
变量C++ 值含义
\lambda0.9衰减因子。90% 保留旧权重，10% 来自新奖励
\psi = \omega_130.0新解 = 全局最优 → 最高奖励
\psi = \omega_210.0新解优于当前解
\psi = \omega_33.0新解被接受（不如当前但还行）
\psi = \omega_40.0新解被拒绝 → 无奖励

5.4 SA 接受准则

用模拟退火决定是否接受"不如当前解"的新解：

P_{accept} = \begin{cases} 1 & \text{if } f(x') > f(x) \text{ (更好)} \\ \exp\left( \dfrac{f(x') - f(x)}{T} \right) & \text{if } f(x') \leq f(x) \text{ (更差)} \end{cases}
参数C++ 默认值含义
T_init50.0初始温度。足够高以允许初期大量探索
alpha_cool0.995几何冷却因子。每轮 T ← 0.995 × T
max_iterations500最大迭代次数
destroy_pct15每次 destroy 移除的节点百分比
early_stop200 轮无改进连续无全局改进则提前终止

7. 复杂度分析

阶段复杂度说明
问题生成O(n^2)距离矩阵计算
GRASP 构造（单次）O(m \cdot n^2 \cdot L)每轮枚举所有 (节点×路线×位置)
2-opt（单路线）O(L^2)枚举所有 (i,j) 对
RelocateO(m^2 \cdot n \cdot L^2)最贵的局部搜索算子
ExchangeO(m^2 \cdot L^2)
ALNS Destroy（单次）O(n \log n)排序/洗牌
ALNS Repair（单次）O(q \cdot m \cdot n \cdot L)q = 被移除节点数（≈ 15% 的已访问节点）
ALNS 总复杂度O(\text{iter} \cdot (m^2 \cdot n \cdot L^2 + \text{repair}))LS 主导，repair 次之

n = 总节点数，m = 队员数，L = 单条路径平均长度（远小于 n），q = 移除节点数

运行结果示例（n=30, m=3, Tmax=200）

求解器得分耗时策略
