1. 问题定义 — TOPTW

TOPTW (Team Orienteering Problem with Time Windows)：m 个队员、n 个地点，各有分数 sᵢ、服务时间 dᵢ、时间窗 [oᵢ, cᵢ]，时间预算 Tmax。目标：最大化总收集分数，每点最多访问一次。

字段类型含义
n, mint总节点数（含 depot）、队员数
Tmaxdouble每队时间预算
depot_start/endint起点/终点节点索引（均为 0）
x, yvector<double>节点坐标
distvector<vector<double>>欧几里得距离矩阵
scoresvector<int>各节点分数；depot = 0
servicevector<double>服务时间；depot = 0
tw_open/closevector<double>时间窗 [o, c]；depot = [0, Tmax]

2. 解的表示 & 解码器

解 = m 条路径，每条是 vector<int> 节点序列。采用间接表达：只存访问顺序，到达时间由解码器递推。

arrive_k = \max(arrive_{k-1} + d_i + dist(i,j),\; o_j)
变量含义
arriveₖ在路径位置 k（节点 j）的到达时间
arriveₖ₋₁ + dᵢ + dist(i,j)自然到达时间
oⱼ节点 j 时间窗开门时间；早到则等待

两条检查：① arrive_k \leq c_j（不晚于关门） ② arrive_k + d_j + dist(j, end) \leq T_{max}（能回终点）

3. GRASP 贪心随机自适应构造

从空路径开始，反复插入未访问节点到性价比最高位置：

ratio(node, team, pos) = \frac{scores[node]}{\max(insertion\_cost,\; 0.01)}

RCL 随机化：α = 0 → 纯贪心；α > 0 → 从前 \alpha \times |candidates| 个候选中随机选。多起点：10 次构造，α 从 0 递增到 0.27。

4. 局部搜索算子

4.1 2-opt（路径内反转）

断边 {i,i+1} 和 {j,j+1}，重连 {i,j} 和 {i+1,j+1}，中间段反转。O(L²)。

4.2 Relocate（跨路径迁移）

从路线 A 移除 v，插入路线 B。O(m²·n·L²)。

4.3 Exchange（跨路径交换）

路线 A 的 a ⇄ 路线 B 的 b 互换。O(m²·L²)。

5. ALNS 框架

ALNS (Adaptive Large Neighborhood Search) 是全局优化框架。核心循环：选 destroy + repair 算子 → 破坏 → 修复 → 局部搜索 → 决定接受/拒绝。

5.1 Destroy 算子

① Random Removal

从已访问节点中均匀随机选出 q 个。最基础、最快的破坏方式。

② Shaw Removal

选一个随机种子，移除与其最相关的 q 个节点。相关性 = 距离相似度（权重 0.6）+ 时间窗重叠度（权重 0.4），加入随机噪声避免每次都一样。

relatedness(v, seed) = 0.6 \cdot \frac{dist(seed, v)}{\max\_dist} + 0.4 \cdot (1 - tw\_overlap) + noise

③ Worst Removal

移除 score / insertion_cost 比值最低的 q 个节点——"性价比最差的"。公式：

ratio(v) = \frac{scores[v]}{dist(prev, v) + dist(v, next) - dist(prev, next)}

5.2 Repair 算子

① Greedy Insertion

每轮选 score / cost 最大的 (node, route, pos) 组合插入。纯贪心，最快。

② Regret-k Insertion

每轮选 regret 最大 的节点先插。Regret = 第 k 优的 cost 之和 − k × 最优 cost。k 越大越保守（默认 k=3）。直觉：不插这个节点会"后悔"，因为它只有少数好位置。

regret(node) = \sum_{r=1}^{k} (cost_{(r)} - cost_{best})

③ Noise Insertion

在贪心插入的基础上，对 cost 施加随机扰动：cost_{noisy} = cost \times (1 + U(-\eta, +\eta))。噪声水平 η 默认 0.15。目的：增加多样性，避免总是插入相同节点。

5.3 元启发式三层

Layer 1 — Simulated Annealing（接受准则）

决定是否接受一个"不如当前解"的新解。Metropolis 准则：

P_{accept} = \begin{cases} 1 & \text{if } f(x') > f(x) \\ \exp(\frac{f(x') - f(x)}{T}) & \text{otherwise} \end{cases}

参数：T₀ = 50.0，冷却因子 α = 0.995（每轮 T ← 0.995·T）。高温 → 大量接受差解（探索）；低温 → 近似贪心（利用）。

Layer 2 — Tabu Search（禁忌记忆）

每个被 Destroy 移除的节点，在 tabu_tenure（默认 12）轮内禁止被重新插入。防止搜索在同一组节点上来回震荡——刚移除的节点马上又插回去等于白做。

参数默认值作用
tabu_tenure12禁忌期长度（迭代次数）
数据结构map<int,int>node → 解禁迭代号

Layer 3 — Adaptive Weights（自适应权重）

每轮轮盘赌选 destroy + repair，执行后根据结果更新权重：

p_j = \frac{\rho_j}{\sum_k \rho_k}, \qquad \rho_{selected} = \lambda \cdot \rho_{selected} + (1-\lambda) \cdot \psi
奖励 ψ条件值
ω₁新解 = 全局最优30
ω₂优于当前解10
ω₃被 SA 接受（差解）3
ω₄被拒绝0

6. 两个求解器 — 并列关系

项目提供两个独立求解器，它们并列运行、互不依赖。main.cpp 对同一个算例分别调用两者，输出对比结果。 两者共享同一套基础设施（§1 问题定义、§2 解码器、§3 构造、§4 局部搜索），但求解策略不同。

求解器 A：Multi-start GRASP + LS

solver.hpp

for restart = 1..10:
    α ← min(0.3, restart × 0.03)
    sol ← construct_greedy(α)    // §3
    sol ← local_search(sol)      // §4
    keep best
return best

策略：10 次独立构造 + 局部搜索，取最优。简单直接。
跳出局部最优：靠不同 α 的随机性

求解器 B：ALNS

alns_solver.hpp

sol ← best from Solver A (initial)
for iter = 1..500:
    d,r ← roulette_wheel(w)      // Layer 3
    sol' ← r(d(sol))             // destroy+repair
    sol' ← local_search(sol')    // §4
    accept? (SA, Layer 1)        // Layer 1
    update_weights(w, result)    // Layer 3
    T ← α × T                    // Layer 1
return best

策略：从 A 的最优解出发，Destroy-Repair + 三层元启发式持续探索。
跳出局部最优：SA 接受差解 + Tabu 防循环 + 自适应选算子

⚡ 两者关系
不是流水线——不是说 A 的结果是 B 的输入步骤。A 和 B 是两个独立的求解策略。
B 的初始解用了 A 的 best（仅此一个连接点），之后 B 完全独立运行 500 轮 ALNS 迭代。
main.cpp 对同一算例调用 A 和 B，输出两者的分数和时间供对比。
如果要只用 B，可以跳过 A，用 GRASP 直接给 B 做初始解（B 内部已包含此逻辑）。

7. 算法复杂度分析

阶段复杂度主导因素
GRASP 构造（单次）O(m \cdot n^2 \cdot L)枚举所有 (node×route×pos)
2-optO(m \cdot L^2)枚举所有 (i,j) 对
RelocateO(m^2 \cdot n \cdot L^2)枚举所有 (src, dst, node, pos)
ExchangeO(m^2 \cdot L^2)枚举所有 (r1, r2, pos1, pos2)
Random/Shaw/Worst DestroyO(n \log n)排序或洗牌
Greedy/Regret/Noise RepairO(q \cdot m \cdot n \cdot L)q 个节点 × 枚举位置
ALNS 每轮O(m^2 \cdot n \cdot L^2)LS 支配，repair 次要
ALNS 总计O(\text{iter} \cdot m^2 \cdot n \cdot L^2)

8. 基准测试结果

⚠ 得分率 ≠ 算法质量 — 先看懂计分规则

表中"总分"列（如 9000、1000）是所有 100 个节点分数之和——不是可达最优解。
受时间预算 T_{max} 的物理约束，你不可能访问全部 100 个节点。

为什么 R 类得分率只有 29~44% 而 C 类有 82~99%？
以 r101 为例：Tmax=230，4 个队员每人分到约 57.5 时间单位。节点平均间距约 40，
服务时间约 10。每人跑一个节点 ≈ 40+10+40=90，连 2 个都跑不完。
4 人 × 7 节点 × 10 分 ≈ 280 分理论上限。我们拿 290 已经超过这个估算。

C 类 Tmax=1236 宽裕得多，算法就能拿到 82~99%。得分率低不是算法差，是时间不够。

29 个标准算例（Righini & Salani 2006），每个 100 客户 + 1 depot。C 类：宽时间窗（Tmax=1236）；R/RC 类：窄时间窗（Tmax=230/240）。每个算例上求解器 A 和 B 分别独立运行。

类型数量mTmax总分得分区间
C (clustered)94123690007380 ~ 8910
R (random)1242301000290 ~ 440
RC (mixed)842401000290 ~ 370

C 类时间窗宽（单个窗口可达 ~1000 单位），每节点分数高（=90），得分率 82%~99%。R/RC 类时间窗极窄（~10 单位），每节点分数低（=10），得分率 29%~44%。

求解器 A（Multi-start GRASP+LS）和求解器 B（ALNS）在 29 个算例上得分一致：总分 81950。两者并列运行于同一算例，B 在三层元启发式的加持下保持了与 A 相同的解质量，同时提供了更强的鲁棒性（在更大算例上可能超越 A）。

Tabu 机制禁 node禁 (node, destroy_op)禁 node 导致可选节点迅速枯竭；禁 move 更精准——只禁止同一算子反复移除同一节点
💡 核心教训
① Tabu 要禁 move 不要禁 node——node-tabu 在 visited node 少的实例上迅速堵死全部选项
② destroy 不宜过大——repair 只是贪心启发式，无法从大规模破坏中恢复最优结构
③ SA 温度要匹配分数尺度——分数差可达几百点时，T=30 不够，需要 T≥100
④ 自适应权重在 200+ 轮后出现明显分化——Shaw Removal + Noise Insertion 是最优组合
🔍 调试时重点关注什么
① tabu blocked 太多（如 4/5 被拦）→ tabu_tenure 太大，降低到 6~8
② LS delta 始终为 0 → 解已经到了局部最优，LS 无法改进 → 尝试增大 destroy_pct
③ accepted_worse ~ 连续出现但 best 不涨 → SA 温度太高，降温太慢 → 降低 α_cool 到 0.98
④ 某个算子权重持续走高 → 自适应在工作，该算子在这个阶段最有效
⑤ 所有权重趋同 → λ 太大（历史权重太顽固）→ 降低 λ 到 0.7~0.8
