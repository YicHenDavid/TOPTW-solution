TOPTW 算法详解

Team Orienteering Problem with Time Windows — 启发式求解器技术报告

1. 问题描述

TOPTW 是一个经典的 NP-hard 组合优化问题：

  • 平面上有 n 个地点，每个地点 i 有：
    • 分数 s_i：访问后获得的收益
    • 服务时间 d_i：在此地停留所需时间
    • 时间窗 [o_i, c_i]：只能在这个时间段内到达
      （早到需等待至开门，晚到则路径不可行）
  • m 个队员，每人从 起点 出发、到达 终点
  • 每人有时间预算 T_{max}，路径总耗时不能超过此值
  • 目标：最大化所有队员收集的总分数
  • 约束：每个地点最多被访问一次（不可重复）
💡 约定
起点和终点视为虚拟节点——分数 = 0、服务时间 = 0、时间窗 = [0, T_{max}]，始终可到达。

2. 整体求解框架：多起点启发式

由于问题是 NP-hard 的，无法在多项式时间内精确求解。本方案采用 Multi-start Heuristic：多次从不同随机起点构造解，每次经局部搜索打磨，最终取最优结果。 

best_solution ← ∅
best_score ← -1

for restart = 0 to n_restarts-1:
    α ← min(0.3, restart × 0.03)        ╱╱ 随机性逐步增大
    rng ← new RNG(seed = restart × 137 + 1)

    sol ← construct_greedy(rng, α)       ╱╱ 第3节 — GRASP 构造初始解
    score ← solution_score(sol)

    for iter = 0 to max_ls_iter-1:       ╱╱ 第5节 — 局部搜索改进
        prev ← score
        for each route in sol:
            route ← two_opt(route)        ╱╱ 路径内 2-opt
        sol ← relocate(sol)              ╱╱ 跨路径迁移节点
        sol ← exchange(sol)              ╱╱ 跨路径交换节点
        score ← solution_score(sol)
        if score ≤ prev: break            ╱╱ 无改进则提前终止

    if score > best_score:
        best_solution ← sol
        best_score ← score
🔑 核心思想
单次构造+搜索可能陷入局部最优。通过多次重新构造（每次 α 不同，随机性递增）， 在开发（exploitation）与探索（exploration）之间取得平衡。

3. 贪心随机自适应构造 (GRASP)

3.1 基本贪心插入

从"空路径"（每队只有 [起点, 终点]）开始， 反复将未访问节点插入到使性价比最高的位置： 

ratio(node, team, pos) = score[node] / max(insertion_cost, 0.01)

其中 insertion_cost = 插入该节点后，路径终点到达时间的增量（详见 §4.3）。

单轮贪心：遍历所有 (未访问节点 × 队伍 × 插入位置)，选 ratio 最大者插入，直到没有可插节点为止。

3.2 RCL 随机化

纯贪心每次构造出相同解，与多起点框架不兼容。引入参数 \alpha \in [0,1]：

α 值行为
\alpha = 0纯贪心 —— 直接选 ratio 最高的候选
\alpha > 0RCL (Restricted Candidate List)：按 ratio 降序排列后，
从前 RCL_size = max(1, α × |candidates|) 个候选中随机选一个

每次重启时 α 递增：α = min(0.3, restart × 0.03)，即从 0 逐步增大到 0.3。

3.3 时间窗可行性检查

每个候选插入操作需通过两个硬性检查（详见 §4.2）：

  1. 到达时间 ≤ 节点时间窗关闭时间 c_j
  2. 服务完该节点后，仍能在 T_{max} 内赶到终点

不满足任一条件的候选直接丢弃，不进入 candidates 列表。

# 伪代码
def construct_greedy(prob, rng, alpha):
    solution = [[start, end] for _ in range(m)]
    unvisited = {1, 2, ..., n-2}

    while unvisited:
        candidates = []
        for node in unvisited:
            for team in 0..m-1:
                route = solution[team]
                for pos in 0..len(route)-2:       # 遍历所有可插入位置
                    cost = compute_insertion_cost(route, node, pos)
                    if cost < ∞:
                        ratio = score[node] / max(cost, 0.01)
                        new_route = route[:pos+1] + [node] + route[pos+1:]
                        candidates.append((ratio, team, node, new_route))

        if not candidates: break

        sort candidates by ratio descending

        if alpha == 0:
            pick = candidates[0]
        else:
            rcl_size = max(1, int(len(candidates) * alpha))
            pick = candidates[ random_int(0, rcl_size-1) ]

        solution[pick.team] = pick.new_route
        unvisited.remove(pick.node)

    return solution

4. 路径评估 (Route Evaluation)

4.1 到达时间递推

沿路径依次推算每个节点的到达时间：

arrive_k = \max\bigl(arrive_{k-1} + d_{i} + \operatorname{dist}(i, j),\;\; o_j\bigr)

其中 i = route[k-1]，j = route[k]，初始 arrive_0 = 0。

  • 括号第一项：自然到达时间（上一节点的到达 + 服务 + 旅行）
  • 括号第二项：时间窗开门时间 o_j，若早到需原地等待

4.2 两种不可行情况

判定条件含义
时间窗违规 arrive_k > c_j 到达时窗口已关闭，不可行
回不来 arrive_k + d_j + \operatorname{dist}(j, end) > T_{max} 服务完毕后无法在预算内赶到终点

4.3 插入成本

\operatorname{cost} = arrive'_{end} - arrive_{end}

即插入节点后，路径终点到达时间的增量。若插入导致路径不可行，cost = +∞。

5. 局部搜索算子 (Local Search)

三个算子按顺序迭代执行，直到总分不再提升（每轮都取 first-improvement 而非 best-improvement）。

5.1 2-opt — 路径内翻转

目的：通过翻转路径中的一段，减少到达终点的时间（从而为后续插入腾出 slack）。

操作：选 i < j（不包含起点终点），将 route[i..j] 反转。

翻转前 [0] → A → B → C → D → [end]  i=2   j=4 
翻转后 [0] → A → D → C → B → [end]

效果：十字交叉路径中尤为有效——可能消除绕路，缩短总时间。 枚举所有 (i, j) 对，保留使终点到达时间最短的解。 复杂度：O(L^2)（L 为路径长度）。

5.2 Relocate — 跨路径迁移

目的：将一个节点从一条路径移到另一条路径，提高总分。

操作：从路线 A 移除节点 v，插入到路线 B 的某个位置。

移出 A: [0] → X → v → Y → [end] → [0] → X → Y → [end]
移入 B: [0] → P → Q → [end] → [0] → P → v → Q → [end]

效果：释放 A 的时间（可能后续插入更高分节点），同时 B 利用空闲时间"收编"该节点。 复杂度：O(m^2 \cdot n \cdot L^2)。

5.3 Exchange — 跨路径交换

目的：交换两条路径中的各一个节点。

操作：路线 A 的节点 a 与路线 B 的节点 b 互换。

A: [0] → X → a → Y → [end] → [0] → X → b → Y → [end]
B: [0] → P → b → Q → [end] → [0] → P → a → Q → [end]

效果：两个节点"各得其所"——各自更适合对方路径的地理位置和时间窗约束。 复杂度：O(m^2 \cdot L^2)。
