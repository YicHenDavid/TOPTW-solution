1. 基础：组合优化与计算复杂性
Combinatorial Optimization & Computational Complexity

1.1 经典组合优化问题

问题描述关键约束复杂性
TSP
Traveling Salesman Problem		 给定城市与距离，访问每个城市恰好一次，回到起点，求最短哈密尔顿回路 每个城市一次，形成闭合回路 NPC
CVRP
Capacitated Vehicle Routing Problem		 多车辆从配送中心出发，服务客户后返回；每车有容量限制 容量约束 + 每个客户仅访问一次 NPC（两层决策：划分+TSP）
VRPTW
Vehicle Routing Problem with Time Windows		 在CVRP基础上，每个客户有服务时间窗 [e_i, l_i] 早到等待，晚到不可行 NPC（比CVRP更难）
BPP
Bin Packing Problem		 n个物品，m个容量W的箱子；最少箱子装下所有物品 每个箱子容量 ≤ W NPC
SCLP
Single Container Loading Problem		 长方体货物装入单个集装箱，可旋转，不交叠，最大化利用率 三维空间约束 NPC
TOPTW
Team Orienteering Problem with Time Windows		 多队员路径问题；每个地点有分数+时间窗+服务时间，时间预算有限 最多访问一次 + 时间窗 + 预算 NPC

1.2 计算复杂性：为什么需要启发式？

TSP 枚举的困境：n=100 个城市，可行解数量为 (n-1)! \approx 10^{158}。

📐 数量级对比
宇宙原子数 ≈ 10^{80}，普朗克时间 ≈ 5.39\times10^{-44}s
假设 10^{80} 个CPU、每个解需 1 个普朗克时间：
枚举所有解仍需 5.39\times 10^{34}s，远超宇宙年龄（4.33\times 10^{17}s）

多项式时间归约：A \leq_p B 读作"A 不比 B 难"——若 B 可多项式求解，则 A 也可；若 A 不可多项式求解，则 B 也不可。

TSP → CVRP → VRPTW 构成层层归约的关系：TSP 是 CVRP 的特殊情况（一辆车、无容量约束），CVRP 是 VRPTW 的特殊情况（去除时间窗）。

1.3 P、NP、NPC、NP-hard
Polynomial / Nondeterministic Polynomial / NP-Complete / NP-hard

类定义示例
P存在多项式时间算法求解最短路、匹配、线性规划、max-flow、指派
NP解可在多项式时间内验证TSP、SAT、3-COLOR、HAM-PATH
NPCNP 中「最难」的问题：任何 NP 问题都可归约到它TSP、SAT、3-SAT、CLIQUE、装箱
NP-hard至少与 NPC 一样难（不要求在 NP 中）许多优化问题的判定版本
🏆 千禧年难题：P = NP ?
悬赏 $1,000,000。学界普遍相信 P ≠ NP。若 P = NP，所有 NPC 问题都将存在多项式算法——目前无人能证明或证伪。

启发式算法的定位：面对 NP-hard 问题，启发式/元启发式放弃「最优解保证」，换取「有限时间内找到足够好的解」。

2.2 间接表达与解码 (Encode–Decode)

核心思想：不直接表达完整解，而是表达一个"间接解"，通过解码器恢复完整解。

间接解空间  —Decoder→  完整解空间

TSPTW 示例：

  • 直接表达：同时记录访问顺序 和 每点的到达时间
    7:30am A → 8:45am E → 11:00am C → ...
  • 间接表达：只记录访问顺序 A E C B F D
    解码器通过 O(n) 递推自动计算到达时间、验证时间窗

条带装箱 (Strip Packing) 示例：

  • 间接解 = 物品的排序 L = (p_1, p_2, ..., p_n)
  • 解码器 = 「最下最左优先」构造启发式：按顺序逐个放入，尽量靠下、靠左

编码-解码的权衡：

  • 多个间接解 → 同一个完整解（存在冗余）
  • 解码器可以是非确定的
  • 某些完整解可能没有对应的间接解 → 可能错过全局最优
  • 解码器通常在多项式时间内解决一个组合子问题（如最短路径、网络流、指派）

2.3 TOPTW 的解的表达

TOPTW 的解直接表达为 m 条路径的集合 + 未访问节点集合：

r_1 = \{0, 1, 3, 5, 0\}
r_2 = \{0, 2, 4, 6, 8, 0\}
u = \{7, 9, 10\}（未访问节点）

这是我们在 C++ 实现中采用的"序列表达"——只存访问顺序，通过解码器计算到达时间。

🔍 TOPTW 解码器 — 到达时间递推详解
变量C++ 类型含义
route vector<int> 一条路径：节点索引序列。如 [0, 3, 7, 31] 表示 depot_start → 节点3 → 节点7 → depot_end
k int 路径中的位置序号（从 0 开始）。route[k] 是第 k 个被访问的节点，k=0 永远是 depot_start
i, j int 相邻两个节点：i = route[k-1]（上一个节点），j = route[k]（当前节点）
arriveₖ double 在路径位置 k 处、节点 j 的实际到达时间（如果早到，需等到开门）
dᵢ prob.service[i] 节点 i 的服务时间（在此地必须停留的时长）
dist(i,j) prob.dist[i][j] 从节点 i 到节点 j 的旅行时间（= 欧几里得距离，预计算在距离矩阵中）
oⱼ prob.tw_open[j] 节点 j 的时间窗开门时间。早于这个时间到达必须等待
cⱼ prob.tw_close[j] 节点 j 的时间窗关门时间。晚于这个时间到达 = 路径不可行
Tmax prob.Tmax 时间预算（每个队员的上限）。整条路径从出发到回到终点的总耗时 ≤ Tmax
depot_end prob.depot_end 终点节点索引（固定为 n-1）。路径最后一站必须是此节点

核心公式拆解

arrive_k = \max\bigl(\; \underbrace{arrive_{k-1} + d_i + dist(i,j)}_{\text{自然到达时间}},\;\; \underbrace{o_j}_{\text{开门时间}} \;\bigr)
情况 A：准点或晚到
如果 arrive_{k-1} + d_i + dist(i,j) ≥ o_j
→ arrive_k = arrive_{k-1} + d_i + dist(i,j)
不用等，到达即开始服务
情况 B：早到
如果 arrive_{k-1} + d_i + dist(i,j) < o_j
→ arrive_k = o_j
到了但门没开，原地等待到开门

具体演算示例

假设路线 [0, 3, 7, 31]，每个节点的数据如下：

节点角色scoreservice dtw_open otw_close c
0depot_start000200
3客户254157187
7客户26372115
31depot_end000200
Step 1 — k=1，从节点 0 到节点 3
i=0, j=3, arrive₀=0, d₀=0, dist(0,3)=sqrt((0-95)²+(50-29)²)≈97.2, o₃=157
自然到达 = 0 + 0 + 97.2 = 97.2  |  开门时间 o₃ = 157
arrive₁ = max(97.2, 157) = 157  ← 早到了！等到 157
✓ 检查：157 ≤ c₃=187 ✓  |  Slack: 157+4+dist(3,31)≈157+4+5.9=166.9 ≤ 200 ✓
Step 2 — k=2，从节点 3 到节点 7
i=3, j=7, arrive₁=157, d₃=4, dist(3,7)=sqrt((95-64)²+(29-97)²)≈75.6, o₇=72
自然到达 = 157 + 4 + 75.6 = 236.6  |  开门时间 o₇ = 72
arrive₂ = max(236.6, 72) = 236.6
✗ 检查：236.6 > c₇=115  → 不可行！时间窗关闭

注意：Step 2 中虽然 3→7 距离很远（自然到达 236.6），但因为早到的那段路没赶上时间窗，最终还是晚了。这个例子说明不是所有节点顺序都能形成可行路径——这就是为什么构造时要逐个检查可行性。

两条硬性可行性检查

检查 ① 时间窗违规
arrive_k \leq c_j
到达节点 j 的时间 不能晚于 其关门时间。
如果 arriveₖ > cⱼ → 路径直接判为不可行（return nullopt）。
对应 C++: if (arrive[k] > prob.tw_close[j]) return std::nullopt;
检查 ② 回不到终点
arrive_k + d_j + dist(j, end) \leq T_{max}
在节点 j 服务完后，必须还能在时间预算内赶到终点。
即使时间窗本身没问题，路途太远也可能让后续行程超时。
对应 C++: if (arrive[k] + prob.service[j] + prob.dist[j][prob.depot_end] > prob.Tmax) return std::nullopt;

C++ 实现对照

// evaluate_route() — 核心解码器 (evaluate.hpp)
std::optional<std::vector<double>> evaluate_route(
    const std::vector<int>& route, const Problem& prob) {

    if (route.size() < 2) return std::nullopt;  // 空路径无效

    std::vector<double> arrive(route.size(), 0.0);  // arrive[0] = 0

    for (size_t k = 1; k < route.size(); k++) {
        int i = route[k - 1];          // 上一个节点
        int j = route[k];              // 当前节点

        // 核心递推: 自然到达 vs 开门时间
        double candidate = arrive[k-1] + prob.service[i] + prob.dist[i][j];
        arrive[k] = std::max(candidate, prob.tw_open[j]);

        // 检查①: 时间窗
        if (arrive[k] > prob.tw_close[j]) return std::nullopt;

        // 检查②: 能否赶到终点
        double slack = arrive[k] + prob.service[j] + prob.dist[j][prob.depot_end];
        if (slack > prob.Tmax) return std::nullopt;
    }
    return arrive;  // 返回所有到达时间（可行）
}

返回值：如果路径可行 → std::optional 包裹到达时间数组；如果不可行 → std::nullopt。
调用方通过 .has_value() 判断可行性，通过 .value() 取到达时间

3. 构造启发式 (Constructive Heuristics)

🎯 这层的角色：构造 (Construction)
构造启发式从零搭建一个完整解，不改进已有解、也不管全局搜索策略。
「一次成型」——做完就不反悔。通常作为 §4 局部搜索的起点和 §5/§6 元启发式的子模块。

构造式启发法逐步构建一个解，每个决策只考虑"当前最佳"而不回溯（贪心/短视 myopic）。属于 一次成型 的算法。

📐 什么是 TSP / VRP 的构造算法？

TSP 构造算法

输入：n 个城市 + 两两之间的距离矩阵
输出：一条访问所有城市恰好一次并回到起点的回路（Hamiltonian cycle）
过程：从一个或两个城市开始，每次选一个未访问城市插入到回路的某个位置，直到所有城市都在回路中

VRP 构造算法

输入：1 个 depot + n 个客户 + 距离矩阵 + 车辆容量
输出：m 条路径，每条从 depot 出发、回到 depot，满足容量约束
过程：先为每个客户建独立路径（或一组初始路径），再按某种规则合并/调整路径

构造 vs 改进 — 两种启发式的分工

构造 (Constructive)改进 (Improvement)
输入空解（或只有起点终点）已有的完整解
输出一个完整可行解一个更好的完整可行解
典型操作逐个插入节点、逐步扩展路径2-opt 翻转、重定位、交换节点
角色生成初始解（为后续优化提供起点）打磨优化解（消除局部缺陷）
TSP 例子Nearest Neighbor、Cheapest Insertion2-opt、3-opt
VRP 例子Saving Heuristic、SweepRelocate、Exchange、2-opt*

构造算法的共同模式：贪心选择 + 顺序决策 + 不回溯。每次选"当前看起来最好"的节点/边/合并操作，做完就不反悔。一旦完整解构造完毕，后续由改进算法（局部搜索）继续打磨。

3.1 TSP 的构造算法

最近邻法 (Nearest Neighbor) — O(n²)

1. 随机选择起始节点 v₁, 初始化回路 T = (v₁, v₁), k = 1
2. 重复直到 k = n:
   2.1 选择离 vₖ 最近的未访问节点 vₖ₊₁
   2.2 在 vₖ 之后插入 vₖ₊₁, 更新 T
3. 输出回路 T

最小成本插入 (Cheapest Insertion) — O(n²)

贪心策略：每次选择 插入成本增量最小 的未访问节点和插入位置：

cost(i,j,v) = c_{iv} + c_{vj} - c_{ij}
变量含义
i, j当前回路 T 中一条弧的两个端点，即插入位置的前驱和后继
v待插入的未访问节点
cab从节点 a 到节点 b 的距离（旅行成本）
civ从 i 到新节点 v 的新增距离
cvj从新节点 v 到 j 的新增距离
cij原来从 i 直接到 j 的距离（可省去）
cost(i,j,v)插入 v 到 (i,j) 之间的净增加成本 = 两条新边 − 一条旧边
\arg\min_{(i,j) \in A(T),\; v \in V\setminus T}\; cost(i,j,v)
符号含义
A(T)当前回路 T 中所有弧的集合。每条弧 (i,j) 代表一个可插入位置
V \ T尚未被加入回路的节点集合（V = 全部节点，T = 已在回路中的节点）
arg min返回使 cost 最小的 (弧, 节点) 组合，即「插入到哪条弧之间、插入哪个节点」

最远插入法 (Farthest Insertion) — O(n³)

  1. 初始化：选相距最远的两个节点构建初始回路 T=(u,v,u)
  2. 选节点：选离当前回路中任意节点 最远 的未访问节点
    v^* = \arg\max_{v \in V\setminus T}\; \min_{u \in T}\; c_{uv}
🔍 内层 min cuv 详解 — 为什么叫"最远插入"？

理解这个公式的关键是从右往左读——先算内层 min，再算外层 arg max：

第 1 步 — 内层 min：计算 v 到回路的「最近距离」
d(v) = \min_{u \in T}\; c_{uv}

对每个未访问节点 v，遍历回路中所有已有节点 u，取 v 到 u 的距离的最小值。
这个值回答：「节点 v 离当前回路有多近？」——距离越小，v 越靠近回路。

第 2 步 — 外层 arg max：选"离回路最远"的 v
v^* = \arg\max_{v \in V\setminus T}\; d(v)

有了每个 v 的 d(v)，取 d(v) 最大 的那个 v 作为 v*。
这个值回答：「哪个节点离回路最远？」——优先把最远的节点圈进来。

    Look-ahead评估后续 k 层决策的影响后再选择当前决策（前瞻树搜索）
    Racing同时尝试多条构造路径，按固定间隔筛选有潜力的路径继续推进
    Learning通过记忆（信息素矩阵）优先选择历史上导向高质量解的决策（ACO (Ant Colony Optimization) 的核心）
    🔗 构造法在元启发式中的角色
    单独使用构造法很难产生高质量解。但它作为子模块至关重要： 生成初始解（ILS、VNS 的起点）、解码器（遗传算法中从基因型恢复表现型）、 修复算子（LNS 中的 Recreate 环节）。

    4. 局部搜索 (Local Search)

    🎯 这层的角色：改进 (Improvement)
    局部搜索不构造新解，而是对一个已有的完整解做局部调整，试图找到更好的邻居。
    它只负责"在当前位置附近找更好的"，不负责"从零搭建解"（那是 §3 构造的事）
    也不负责"跳出局部最优的全局策略"（那是 §5 元启发式的事）。

    邻域 Neighborhood：映射 N: S \to 2^S，对每个解 s 定义其邻居解集合 N(s)。搜索空间可建模为有向图 G=(V,A)，每个节点代表一个解，每条弧代表一个邻居关系。

    s ← s₀  // 初始解

    s ← Select( N(s) ) // 选择一个改进的邻居
return s  // 返回局部最优解

    🔧 Or-opt — 整体搬迁连续节点（不反转）

    Or-opt 是什么？

    Or-opt 将路径中 1、2 或 3 个连续节点 整体取出，插入到另一个位置。

    由 I. Or 在 1976 年提出，本质是 3-opt 的一个子集——只执行「搬迁」类操作，不执行「反转」类操作。

    📊 为什么复杂度是 O(n²) / O(n³)？

    路径内算子的复杂度来自枚举所有可能的操作位置。对一条长度为 L 的路径，每个候选位置都要尝试、都要调用 O(L) 的 evaluate_route() 验证可行性。

    2-opt — O(L²)

    两个断开位置 i 和 j 需要满足 1 \leq i < j \leq L-1（不包含起点 0 和终点 L−1）。
    枚举所有 (i, j) 对：

    for i = 1 to L-3:    // 第一个断点（i 后面是 i+1）
      for j = i+1 to L-2:  // 第二个断点（j 后面是 j+1）
        尝试 reverse(i+1, j) + evaluate_route()  // O(L)

    总枚举量 ≈ ½(L−2)(L−3) = O(L²) 个 (i,j) 组合，每个 evaluate 是 O(L) → O(L³) naive
    → 但可增量评估降到 O(1) per check → 总 O(L²)

    3-opt — O(L³)

    三个断开位置 i, j, k，满足 1 \leq i < j < k \leq L-1。枚举所有三元组：

    for i = 1 to L-3:
      for j = i+1 to L-2:
        for k = j+1 to L-1:
          尝试 7 种重连 + evaluate_route()

    总枚举量 ≈ (L−2)(L−3)(L−4)/6 = O(L³) 个 (i,j,k) 组合
    这就是为什么 3-opt 比 2-opt 贵一个数量级

    Or-opt — O(L²)

    枚举片段起点 × 片段长度（1/2/3）× 目标插入位置：

    for seg_start = 1 to L-2:      // 片段起点
      for seg_len = 1 to min(3, L-seg_start-1):  // 长度 1/2/3
        for target = 0 to L-seg_len-1:    // 目标位置
          如果 target 在片段内则跳过
          尝试搬迁 + evaluate_route()

    枚举量 ≈ L × 3 × (L−1) ≈ O(L²)

    汇总

    算子枚举量本质
    2-opt组合数 C(L−2, 2) ≈ L²/2遍历所有两位置对 (i, j)
    Or-optL × 3 × (L−1) ≈ 3L²遍历所有片段起点 × 长度 × 目标位
    3-opt组合数 C(L−2, 3) ≈ L³/6遍历所有三位置组 (i, j, k)

    路径长度 L 通常远小于总节点数 n（因为每个队员只访问部分节点），所以路径内算子的实际开销比看上去小。
    TOPTW 项目中，m 个队员各有 m 条路径，总复杂度要 × m。

    技术说明
    O(1) 增量评估维护辅助数据（如路径总长、最早出发时间、最晚到达时间），增量计算 \Delta = f(s_{new}) - f(s_{old}) 而非重新计算全解

    5. 轨迹式元启发式 (Trajectory-based)

    「轨迹式」指的是算法在搜索空间中从一个解移动到另一个解，形成一条搜索轨迹。核心挑战：如何跳出局部最优。

    🏗️ 算法分工 — 构造 vs 改进 vs 全局优化框架

    前面几节讲了很多"算法"，但它们不是平级的。按角色可以分为三层：

    🧱
    构造启发式

    角色：生成初始解
    干什么：从零搭建一个
    完整可行解
    输入：空解 / 部分解
    输出：一个完整可行解
    例子：Nearest Neighbor、
    Cheapest Insertion、
    GRASP (Greedy Randomized
    Adaptive Search Procedure)、
    Saving

    🔧
    局部搜索算子

    角色：单步改进
    干什么：对已有解的局部
    做调整，找更好的邻居
    输入：一个完整可行解
    输出：一个更好的
    （或同样的）可行解
    例子：2-opt、Or-opt、
    Relocate、Exchange

    🧠
    元启发式框架

    角色：全局搜索指挥
    干什么：调度构造+局部搜索
    决定何时扰动、何时
    接受差解、何时切换策略
    输入：问题实例
    输出：全局最优（近似）
    例子：ILS、VNS、SA、
    Tabu Search、LNS、GA

    轨迹式元启发式的本质

    轨迹式元启发式不管构造（构造由专门的构造启发式完成），也不亲自做局部搜索（局部搜索由 2-opt/relocate 等算子完成）。 它的工作是"指挥"：在什么时机、以什么方式调用构造和局部搜索，让搜索过程不至于卡在局部最优。

    框架核心策略依赖的构造方法依赖的局部搜索
    ILS Perturbation 扰动 → LS → 重复 GRASP (Greedy Randomized Adaptive Search Procedure) / random init（生成初始解和修复被扰动解） 2-opt + relocate + exchange（精细化搜索）
    VNS 切换不同邻域结构 同 ILS 多种邻域算子轮换（N₁=2-opt, N₂=relocate, …）
    LNS / ALNS Destroy → Repair（Repair = 构造） Repair 环节直接调用构造启发式 可选的 LS 精化（destroy 前或 repair 后）
    SA 概率接受差解（Metropolis 准则） 生成初始解（一次） 随机选一个邻域算子，评估后按概率接受
    Tabu Search 禁忌列表 + 藐视准则 生成初始解（一次） 遍历受限邻域找最优邻居

    一句话总结：构造 = 盖房子、局部搜索 = 装修房子、元启发式 = 指挥整个建造+装修过程，确保最终房子质量最高。
    轨迹式元启发式属于「全局优化框架」，不是构造，也不是单一的改进算子。

    5.1 ILS — 迭代局部搜索
    Iterated Local Search

    问题：为什么需要 ILS？

    局部搜索（§4）的问题在于：它沿着下降方向走，一旦到达局部最优（所有邻居都不如当前解），就停住了。
    最简单的补救是 Multi-start：随机生成一个新解，再跑一次局部搜索，重复 N 次取最优。
    但 Multi-start 有一个致命缺点：每次从零开始，完全抛弃了之前搜索积累的好结构。

    ILS 的想法：不是从零重启，而是对当前局部最优解做有限的破坏（Perturbation），
    保留大部分好结构，仅打乱一小部分——然后在这个"半新"的解上再跑局部搜索。

    ❌ Multi-start：每次从零开始
    restart 1: 随机解 → LS → 局部最优 A
    restart 2: 随机解 → LS → 局部最优 B
    restart 3: 随机解 → LS → 局部最优 C
    每次完全随机，之间毫无关联
    →
    ✅ ILS：每次从前一个局部最优变异而来
    随机解 → LS → 局部最优 A
    A → Perturb → 半新解 A' → LS → 局部最优 B
    B → Perturb → 半新解 B' → LS → 局部最优 C
    B 继承了 A 的好结构，C 继承了 B 的好结构

    Perturbation 大小的校准 — ILS 最关键的设计

    Perturbation 的大小直接影响 ILS 的效果。以 TSP 为例，扰动 = 随机 swap 多少次：

    扰动程度例子（100 个节点的 TSP）效果结果
    太小 swap 1 次 s' 几乎 = s*，LocalSearch 直接回到原来的局部最优 白费力气，ILS 退化
    合适 swap 5~15 次 s' 保留 85%+ 的原结构（好边还在），但改变了足够多（可能跳到邻近谷底） 保持好结构的探索
    太大 swap 50 次 s' 几乎 = 随机解，丢失了 s* 中所有好的边和节点顺序 退化 = Multi-start
    💡 我们的 TOPTW C++ 实现中的扰动策略
    采用 Multi-start 而非严格 ILS。每次 restart 用不同的随机种子和 α 重新构造 + LS。
    这等价于 ILS 的 Perturbation = 完全随机重构（扰动过大）。升级为 ILS 可以保留 best sol 的部分结构后做局部破坏再 LS。

    接受准则 (Acceptance Criterion)

    ILS 的接受准则决定是否把新找到的局部最优 s*' 作为下一轮的起点：

    策略逻辑适用场景
    Always accept 无论 s*' 比 s* 好还是差，都接受为新起点 最常用。配合"始终记住 global best"即可
    Better only 只接受更好的 s*' 保守，但可能导致搜索停滞（无法接受差解换盆地）
    SA-style 更好的直接接受，更差的以概率 exp(−Δ/T) 接受 融合 SA 思想，温度逐步降低

    5.2 VNS / VND — 变邻域搜索
    Variable Neighborhood Search / Variable Neighborhood Descent

    两个核心引理：

    1. 一个邻域结构的局部最优，未必是另一个邻域结构的局部最优

    核心思想：不断切换邻域结构，因为在一个邻域中陷入局部最优后，换一个邻域可能还能改进。

    假设有 4 个邻域：N₁ = 2-opt、N₂ = relocate、N₃ = exchange、N₄ = Or-opt（所以 k_max = 4）。
    初始解 s 的目标值 f(s) = 500。

    轮次当前 k发生的事情k 变为s 状态
    1 k=1 在 N₁ (2-opt) 中搜，找到 s'，f(s')=480（比 500 好）
    → 改进成功！    		 k=1 s 更新
    f(s)=480
    2 k=1 在 N₁ 中再搜，f(s')=480（没改进）
    → 改进失败    		 k=2 不变
    3 k=2 在 N₂ (relocate) 中搜，f(s')=450（比 480 好）
    → 改进成功！    		 k=1 s 更新
    f(s)=450
    4 k=1 N₁ 中再搜，无改进 k=2 不变
    5 k=2 N₂ 中再搜，无改进 k=3 不变
    6 k=3 N₃ (exchange) 中搜，无改进 k=4 不变
    7 k=4 N₄ (Or-opt) 中搜，无改进 k=5 不变

    k=5 > k_max=4 → 终止。最终解 f(s) = 450，在所有 4 个邻域中都是局部最优。
    注意观察：每次改进成功后 k 立刻回到 1——因为解变了，便宜的邻域可能又有改进空间了，不浪费时间去试贵的邻域。

    💡 VND 最关键的技巧
    为什么改进后要重置 k=1？
    因为解 s 变了！原来在 N₁ 中没有邻居比旧 s 好，但现在的 s 是新的——N₁ 中可能有邻居比新 s 更好。
    所以回到 k=1 再试，用最便宜的邻域再薅一次。这叫 "一旦改进就回到最小邻域"。

    邻域排序为什么重要？
    便宜的放前面（k 小），贵的放后面（k 大）。每次有改进就回 k=1 → 尽量用便宜的邻域，实在不行再升级。
    如果反过来（贵的放前面），每轮都跑最慢的邻域，效率极低。

  k ← 1
  repeat
    s' ← IterativeImprovement(s, Nₖ)    // 在第 k 个邻域中搜索
    if f(s') < f(s) then
      s ← s'
      k ← 1                              // 改进成功，回到最小邻域
    else
      k ← k + 1                          // 改进失败，切换到下一个邻域
  until k = k_max

    搜索空间图中的"邻居节点"就是 N(s) 中的解

    💡 VNS 的"变邻域"就是"换算子"
    VNS 的 N₁, N₂, N₃... 就是不同的算子（2-opt, relocate, exchange…）。
    "切换邻域" = "改用另一个算子来生成邻居解" = "换一套生成规则"。

    换算子的效果：原来用 2-opt 找不到更好的邻居（卡住了），换用 relocate 可能就能找到——
    因为 relocate 生成的邻居解集合完全不一样。

    邻域排序：从小到大（small → large），先用便宜的邻域，不行再换贵的。

    Basic VNS — Shaking + VND

    🔍 VNS 和 VND 到底有什么区别？

    VND 只管"下降"——在多个邻域里换着搜，只接受更好的解。一旦所有邻域都搜不到改进就停止。
    问题是：VND 最终还是停在局部最优（虽然是对多个邻域都局部最优，比单邻域强，但仍然不是全局最优）。

    VNS 在 VND 前面加了一步 Shaking（震动）——先用 Shaking 随机跳到远处，再让 VND 从那里开始下降。
    这一步随机跳跃让 VNS 有能力跨越屏障，到达 VND 单独到不了的更深谷底。

    VND
    Variable Neighborhood Descent
    k=1 → 在 N₁ 搜 → 有改进回 k=1
    k=2 → 在 N₂ 搜 → 有改进回 k=1
    k=3 → 在 N₃ 搜 → …
    → 所有邻域都搜不到改进 → 停止

    ⚠ 仍然是确定性下降
    换邻域 ≠ 跳出局部最优
    只是扩大了搜索范围

    →
    VNS
    Variable Neighborhood Search
    k=1 → Shaking（随机跳）→ VND → 有改进回 k=1
    k=2 → Shaking（跳更远）→ VND → …
    k=3 → …
    → 所有 k 的 Shaking 都试完 → 停止

    ✅ Shaking = 随机扰动
    用随机性打破确定性
    真正有能力跳出局部最优

    Shaking 到底做了什么？

    Shaking(x, k) 的意思：在解 x 上随机做 k 次操作（用第 k 个邻域算子），产生一个"被震动过"的解 x'。
    k 越大 → 震动越剧烈 → 跳得越远。

    具体例子：假设 N₂ = relocate
    Shaking(x, k=2)：随机选一个节点，随机 relocate 它到另一条路线 → 做 2 次
    Shaking(x, k=5)：随机 relocate → 做 5 次（跳得更远）

    VNS 的完整逻辑

    VNS = 外层循环（换 k 换 Shaking 强度）+ VND 作为内层（做完 Shaking 后的精细化搜索）。
    一句话：Shaking 让你换盆地，VND 让你沉到盆地最深处。

    VNS 伪代码

    Function BVNS(x, k_max, t_max):          // Basic VNS
  while t < t_max:                       // 外层：时间限制内一直跑
    k ← 1
    repeat
      x' ← Shake(x, k)                  // ① Shaking：用 Nₖ 随机扰动 k 次
      x'' ← BestImprovement(x')         // ② 局部搜索（VND / 单一 LS）
                                         //    从 x' 出发，找到局部最优 x''
      if f(x'') < f(x):                 // ③ Neighborhood Change：
        x ← x''                          //    有改进 → 接受，k 回 1（从头试）
        k ← 1
      else:
        k ← k + 1                        //    无改进 → k++（下次震得更狠）
    until k = k_max                      // 所有震动强度都试过
  return best_x                          // 所有 while 迭代中的最优解
    💡 VNS vs VND — 一句话总结
    VND = 多邻域下降：用不同算子换着搜，但只往下走，最终停在局部最优。
    VNS = Shaking + VND：先随机跳（Shaking）打破僵局，再让 VND 在新盆地往下沉。
    VNS 的 Shaking = ILS 的 Perturbation — 两者功能相同（跳出局部最优），只是 VNS 的 Shaking 的强度随 k 递增。

    5.3 LNS / ALNS — 大邻域搜索
    Large Neighborhood Search / Adaptive Large Neighborhood Search

    核心思想：Ruin-and-Recreate —— 摧毁解的一部分，再重新构造。等价于：固定大部分决策变量，在剩余"自由变量"上优化子问题。

    🏗️ LNS — Ruin-and-Recreate 流程
    🧩 当前解 x
    完整可行解
    →
    💥 Destroy
    随机/相关移除
    5%~20% 节点
    →
    🔧 Repair
    贪心/精确/随机
    重新插入被移除节点
    →
    🧩 新解 xᵗ
    可能改善或恶化

    被 Destroy 移除的节点构成「自由变量」，Repair 在这个缩小的子问题上重新优化 

    Algorithm: Large Neighborhood Search
1. 输入可行解 x
2. xᵇ ← x
3. repeat
4.    xᵗ ← r( d(x) )           // destroy then repair
5.    if accept(xᵗ, x) then x ← xᵗ
6.    if c(xᵗ) < c(xᵇ) then xᵇ ← xᵗ
7. until 终止条件
8. return xᵇ

    自适应大邻域搜索 ALNS

    🔍 LNS 和 ALNS 有什么关系？

    LNS 的问题：只有一组 destroy/repair 算子（比如"随机移除 10% + 贪心重插"）。
    但不同算子在不同阶段效果不同——搜索早期随机移除好，搜索后期可能需要"相关移除"来聚焦。
    固定算子 = 无法适应搜索进程。

    ALNS 的改进：准备多组 destroy 和 repair 算子，每轮根据历史表现用轮盘赌选一组来用。
    表现好的算子权重越来越高 → 自动被选更多次 → 搜索策略随进程自适应调整。

    LNS
    Large Neighborhood Search
    1 个 destroy 算子（如"随机移除 10%"）
    1 个 repair 算子（如"贪心重插"）
    ↓
    每轮：destroy → repair → 检查 → 重复
    ↓
    算子固定，无法进化
    →
    ALNS
    Adaptive Large Neighborhood Search
    多个 destroy 算子（随机、相关、最差…）
    多个 repair 算子（贪心、regret-k…）
    ↓
    每轮：轮盘赌选 destroy+repair → 执行 → 评估表现 → 更新权重
    ↓
    好算子权重↑ 差算子淘汰

    ALNS 的核心机制 — 三步走

    第 1 步：准备算子库

    准备两组算子：Ω⁻ = {d₁, d₂, d₃, …}（destroy 算子集合）、Ω⁺ = {r₁, r₂, r₃, …}（repair 算子集合）。
    每个算子初始权重相同（如 ρ = 1），后续根据表现动态调整。

    Destroy 算子操作Repair 算子操作
    Random 随机移除 q 个节点 Greedy 贪心插入：每次插性价比最高的
    Related 移除地理上相近的节点（更容易重组） Regret-k 后悔插入：选"不插入会最后悔"的节点先插
    Worst 移除贡献最低的节点（分数/成本比最小） Random 随机顺序插入
    Cluster 移除一个聚类中的所有节点 GRASP GRASP 随机贪心插入
    第 2 步：轮盘赌选择算子

    每轮迭代，根据权重用轮盘赌选一个 destroy 和一个 repair 算子：

    p_j^- = \frac{\rho_j^-}{\sum_{k \in \Omega^-} \rho_k^-}, \qquad p_j^+ = \frac{\rho_j^+}{\sum_{k \in \Omega^+} \rho_k^+}
    变量C++ 类型含义例子（3 个 destroy 算子）
    Ω⁻ vector<Destroy> 所有 destroy 算子的集合 Ω⁻ = {Random, Related, Worst}，共 3 个
    Ω⁺ vector<Repair> 所有 repair 算子的集合 Ω⁺ = {Greedy, Regret-2}，共 2 个
    j int 某个特定算子的编号（索引） j=0 → Random、j=1 → Related、j=2 → Worst
    ρⱼ⁻ double 第 j 个 destroy 算子的当前权重 ρ₀⁻=5（Random 权重高，表现好）
    ρ₁⁻=3（Related）、ρ₂⁻=2（Worst 表现差）
    ρⱼ⁺ double 第 j 个 repair 算子的当前权重 同 destroy，每个 repair 算子也有自己的权重
    Σ ρₖ⁻ double 所有 destroy 算子权重的总和（分母） Σ = 5 + 3 + 2 = 10
    pⱼ⁻ double 第 j 个 destroy 算子被选中的概率 p₀⁻ = 5/10 = 50%、p₁⁻ = 3/10 = 30%、p₂⁻ = 2/10 = 20%
    💡 公式核心
    pⱼ⁻ 就是「单个算子的权重」除以「所有算子的权重之和」。
    权重大的占的份额大 → 轮盘上占的面积大 → 被指针指到的概率大 → 被选中更多次。
    pⱼ⁺ 同理，用于选择 repair 算子。
    轮盘赌执行过程
    假设 3 个 destroy 算子权重：d₁=5, d₂=3, d₃=2 → 总权重=10
    随机数 r ← random(0, 10)
      r ∈ [0, 5) → 选 d₁（区间宽度=权重）
      r ∈ [5, 8) → 选 d₂
      r ∈ [8, 10) → 选 d₃
    注意：即使 d₃ 权重最小（2），仍有 20% 概率被选中——保留探索性，防止过早锁定在某个算子。
    第 3 步：根据表现更新权重

    执行选中的 destroy+repair → 得到新解 → 根据新解质量给这对算子打分 → 更新权重：

    \rho_a^- = \lambda \rho_a^- + (1-\lambda)\psi, \qquad \rho_b^+ = \lambda \rho_b^+ + (1-\lambda)\psi
    变量C++ 类型含义例子
    a int 本轮被选中的 destroy 算子的编号 如果这轮选的是 Related removal → a=1
    b int 本轮被选中的 repair 算子的编号 如果这轮选的是 Greedy → b=0
    ρₐ⁻ double destroy 算子 a 的当前权重（更新前） ρ₁⁻ = 3.0
    ρₐ⁻ (新) double destroy 算子 a 的更新后权重（公式左边） 见下方计算实例
    λ double 衰减因子（0 < λ < 1），控制「旧权重」和「新奖励」的混合比例 λ = 0.9 → 90% 保留旧权重，10% 来自新奖励
    ψ double 本轮奖励分数。根据新解质量分四档：ω₁/ω₂/ω₃/ω₄ 如果新解是全局最优 → ψ = ω₁ = 30
    λ ρₐ⁻ double 旧权重的保留部分。λ 越大，历史表现越重要 0.9 × 3.0 = 2.7
    (1−λ) ψ double 新奖励的加入部分。1−λ 越大，对近期表现越敏感 (1−0.9) × 30 = 0.1 × 30 = 3.0
    ρₐ⁻ (新) double 旧保留 + 新奖励 = 最终新权重 2.7 + 3.0 = 5.7（从 3.0 涨到 5.7）
    计算实例：Related removal（ρ₁⁻=3.0）这轮被选中，结果找到了全局最优 → ψ = ω₁ = 30
    新权重 = 0.9 × 3.0 + 0.1 × 30 = 2.7 + 3.0 = 5.7
    → 权重从 3.0 涨到 5.7，下次被选中的概率增加了！

    关键：ρ_b⁺ 的更新公式完全相同，只是操作对象是 repair 算子 b。
    只有 a 和 b 被更新——其他未被选中的算子权重保持不变。

    λ 的选择：λ 接近 1（如 0.95）→ 权重变化慢，依赖长期表现，稳定但反应慢；λ 接近 0（如 0.5）→ 快速响应近期表现，但波动大。
    典型值 λ = 0.8~0.9，在稳定和自适应之间平衡。

    ψ 值条件含义
    ω₁ 新解 = 新的全局最优 ✨ 最高奖励 → 这对算子太棒了
    ω₂ 新解比当前解好 高奖励 → 这对算子不错
    ω₃ 新解被接受（不如当前但还行） 小奖励 → 至少有点贡献
    ω₄ 新解被拒绝 无奖励 → 这对算子这次没用

    λ 是衰减因子（0.8~0.95），控制"历史记忆"的强度。λ 大 → 权重变化慢、依赖长期表现；λ 小 → 快速响应近期表现。
    没有被选中的算子权重保持不变——确保好的算子积累优势，差的算子等待被自然淘汰。

    ALNS 伪代码

    Algorithm: Adaptive Large Neighborhood Search
1. 输入可行解 x
2. xᵇ ← x                                 // global best
3. ρ⁻ ← (1,1,…,1);  ρ⁺ ← (1,1,…,1)       // 所有权重初始为 1
4. repeat
5.    选择 d ∈ Ω⁻ 用概率 p⁻               // 轮盘赌选 destroy
6.    选择 r ∈ Ω⁺ 用概率 p⁺               // 轮盘赌选 repair
7.    xᵗ ← r( d(x) )                      // destroy → repair
8.    if accept(xᵗ, x) then x ← xᵗ         // 接受准则
9.    if c(xᵗ) < c(xᵇ) then xᵇ ← xᵗ        // 更新全局最优
10.   更新 ρ_d⁻ 和 ρ_r⁺ 用迭代奖励 ψ       // 只更新被选中的
11. until 终止条件
12. return xᵇ
    💡 LNS vs ALNS — 一句话总结
    LNS = 一种固定的 destroy + repair 方案（"我要怎么破坏、怎么修复"是固定的）。
    ALNS = LNS + 自适应算子选择（"我有好几种破坏和修复的武器，哪个好用我就多用哪个"）。
    ALNS 的核心创新就是让搜索自己学会"什么时候用什么算子"。

    5.4 SA — 模拟退火算法
    Simulated Annealing

    物理类比：高温 → 原子高能态、高重排概率；缓慢降温 → 能量降低、重排概率减小 → 趋向最低能量态（晶体）。

    物理过程优化算法
    能量状态目标函数值 f(x)
    状态变化移动到邻居解
    温度 T控制参数（接受劣解的概率）
    冷却过程温度递减调度

    Metropolis 接受准则：

    p(T, s, s') = \begin{cases} 1 & \text{if } f(s') \leq f(s) \\[6pt] \exp\left(\dfrac{f(s) - f(s')}{T}\right) & \text{otherwise} \end{cases}
    变量含义
    s当前解
    s'随机生成的邻居解
    T当前温度 — T 越高，exp((f(s)−f(s'))/T) 越大 → 接受劣解概率越大
    f(x)目标函数值（最小化问题，越小越好）
    Δ = f(s') − f(s)Δ ≤ 0 → 改进/持平，直接接受；Δ > 0 → 恶化，以概率 exp(−Δ/T) 接受
    exp(−Δ/T)接受劣解概率：Δ 越大（越差）→ 概率越低；T 越大（越热）→ 概率越高
    Algorithm: Simulated Annealing
1. 选择初始解 ω ∈ Ω, 初始温度 T = t₀, k = 0
2. Repeat
3.    m = 0
4.    Repeat  // 每个温度下执行 Mₖ 步
5.        生成邻居 ω' ∈ N(ω)
6.        计算 Δ = f(ω') - f(ω)
7.        if Δ ≤ 0: ω ← ω'            // 改进，接受
8.        else: 以概率 exp(-Δ/T) 接受  // 恶化，概率接受
9.        m ← m + 1
10.   Until m = Mₖ
11.   k ← k + 1;  T ← cooling(T, k)    // 降温
12. Until 终止条件

    冷却调度 (Cooling Schedule)：

    • 线性冷却：t_{k+1} = t_0\,(1 - k/K_{max})
    • 几何冷却（最常用）：t_{k+1} = \alpha \cdot t_k，0 < \alpha < 1
    变量含义
    t₀初始温度 — 通常设得足够高，使初期几乎接受所有 move（≈ 随机探索）
    tₖ第 k 次降温后的温度 — 越低，接受劣解的概率越小
    k当前降温次数（温度轮次计数器）
    Kmax最大降温次数（总温度轮数）
    α几何冷却因子 — 典型值 0.8~0.99。α 越接近 1，降温越慢（搜索越充分，但耗时越长）
    🌡️ 温度对接受概率的影响
    高温 T=1000
    接受率≈99%
    中温 T=100
    接受率≈37%
    低温 T=1
    接受率≈0%

    高温 = 随机搜索（探索）；低温 = 贪心局部搜索（利用）。冷却速率是关键——太快→早熟，太慢→浪费算力

    ⚠ SA 的局限
    SA 可能离开最优解后再也找不到它 → 始终记住并返回 best-so-far 解。
    理论上可证明在无限时间下收敛到全局最优，但现实中只能有限时间运行。

    5.5 TS — 禁忌搜索
    Tabu Search

    核心思想：接受非改进移动以跳出局部最优（像 SA），但通过 记忆机制（Tabu List）防止循环回到刚访问过的解。TS 本质是 动态邻域 方法——在受限邻域 \tilde{N}(x) \subseteq N(x) 中搜索。

    S₀初始解，用构造启发式生成的第一版initial_solution
    S'候选邻居解（读 S-prime），从受限邻域中选出的最优邻居candidate

    1. 构造初始解 S₀
    用构造启发式（如 GRASP）生成一个初始可行解，作为搜索起点
    2. S ← S₀, f* ← f(S₀), S* ← S₀, T ← ∅
    "当前解 = 初始解；全局最优值 = 初始解的值；全局最优解 = 初始解；禁忌列表 = 空的"
    这一行做了 4 件事：① 把初始解设为当前解 ② 记录它的目标值 ③ 同时把它设为"目前找到的最好解" ④ 清空禁忌列表
    3. While 终止条件未满足:
    "只要还没到最大迭代次数 / 时间 / 连续无改进轮数，就一直循环"
    4. 选择 S' ∈ arg min_{S' ∈ Ñ(S)} f(S')
    "查看当前解 S 的所有邻居（除了被禁忌列表禁止的那些），在这些邻居里选出目标值最小的（最好的）那个，叫它 S'"
    ⚠ 即使选出的 S' 比 S 差（目标值更大），也要走这一步——TS 和 LS 的关键区别就在这里：TS 接受非改进的邻居。
    5. if f(S') < f*: S* ← S', f* ← f(S')
    "如果 S' 的目标值比当前全局最优还好（f(S') < f*）→ 更新全局最优解和全局最优值"
    这是单独维护的 global best，不受 Tabu 影响。即使 S' 是 tabu 的，只要比 f* 更好就可以接受（藐视准则）
    6. S ← S'
    "把 S' 设为新的当前解 S"——不管 S' 是否更好，都移动过去。这就是 TS 跳出局部最优的关键
    7. 将当前 move 记录到 Tabu List T (删除最旧条目)
    "把刚才的操作（如 swap(1,2) 或 relocate(3→7)）放进禁忌列表。列表满了就把最旧的删掉（FIFO）"
    存放的不是完整的解，而是操作的描述——防止短时间内做同样的操作又回到刚才的解
    8. Return S*
    循环结束，返回整个搜索过程中找到的最优解 S*（不是最后的当前解 S）

    6. 种群式元启发式 (Population-based)

    🎯 这层的角色：全局优化框架（同 §5）
    种群式元启发式同样是全局优化框架——不亲自构造、不亲自局部搜索。
    区别在于它维护一组解（种群）而非单个解，通过交叉/变异/选择驱动搜索。
    与 §5 轨迹式的关系：两者可以结合（如 Memetic = GA + LS），也可以各自独立使用。

    与轨迹式不同，种群式算法维护 一组解（种群），通过组合和变异生成新解，模拟自然进化。

    6.1 GA — 遗传算法
    Genetic Algorithm

    Darwin 范式：繁殖 → 竞争 → 选择。优胜劣汰、适者生存。

    核心概念：

    生物学概念算法映射说明
    个体 (Individual)一个可行解
    种群 (Population)一组候选解的集合维持多样性
    染色体 (Chromosome)解的编码表示如二进制串、排列
    适应度 (Fitness)解的质量f(x)，决定生存概率
    基因 (Gene) / 等位基因 (Allele)编码的一部分及其取值解的一个决策变量

    遗传算子

    选择 (Selection) — 两种主流方式：

    方法机制
    轮盘赌 (Roulette Wheel) 每个个体按照适应度占轮盘面积比例被选中：p_i = \frac{f(i)}{\sum_j f(j)}
    f(i) = 个体 i 的适应度（目标函数值）；分母 = 种群所有个体适应度之和；pᵢ = 个体 i 被选中的概率
    锦标赛 (Tournament) 随机选 q 个个体，取其中适应度最高的（如 q=6 选前 2 作为双亲）

    交叉 (Crossover) — 模拟遗传物质交换：

    🧬 单点交叉 (One-Point Crossover)
    Parent A ABC DE
    Parent B FGH IJ
    Child 1 ABC IJ
    Child 2 FGH DE

    随机选一个切点，交换切点后的基因片段。蓝色 = Parent A 来源，黄色 = Parent B 来源，绿色 = 交叉后组合

    变异 (Mutation) — 维持多样性，防止早熟：

    类型操作示例
    Swap交换两个位置1 5 3 4 2 6 → 1 2 3 4 5 6
    Insertion将一个元素移到另一位置1 2 3 4 5 6 → 1 5 2 3 4 6
    Inversion反转一段子序列1 5 4 3 2 6 → 1 2 3 4 5 6
    Displacement将一段子序列移到另一位置
    🧬 排列编码的变异操作示例
    Swap 153426 → 123456
    Inversion 154326 → 123456

    Swap：交换位置 2 和 5。Inversion：反转位置 2~5 的子序列。红色 = 被操作区域，绿色 = 结果

    ⚠ 关键参数
    变异率必须很低 — 大多数变异是破坏性的，过高的变异率会摧毁优良基因。 
    Algorithm: Genetic Algorithm
1. 初始化种群 sp
2. while 终止条件未满足:
3.    通过选择(crossover)生成 spr  ← 交叉
4.    通过变异(mutation)从 sp 和 spr 生成 spm
5.    从 sp、spr、spm 中选择新种群 sp  ← 选择 + 精英保留
6. return 种群中最优个体

    🧬 TSP 的遗传算法 — 编码 / 交叉 / 变异详解

    ① 编码 — 为什么 TSP 用排列而不是二进制？

    TSP 的解是一条访问顺序。二进制编码（如 1 0 0 1 …）适合"选或不选"的问题（如背包），
    但对 TSP 很别扭——你怎么用 0/1 表示"先访问北京、再访问上海、最后去广州"？
    排列编码直接用一个数字序列表示访问顺序，每个数字=城市编号，出现一次且仅一次。

    示例
    8 个城市，编号 1~8。
    染色体 = (3, 5, 7, 2, 1, 6, 4, 8)
    含义：从 3 出发 → 去 5 → 去 7 → 去 2 → 去 1 → 去 6 → 去 4 → 去 8 → 回到 3
    （回路自动闭合，最后访问的 8 与起点 3 之间的边也算距离）

    ② 交叉 — 排列编码的交叉不能简单地"切+粘"

    普通 GA 的交叉（切两半、交换后半段）对排列编码不适用——会产生重复或缺失的城市。
    TSP 需要专门的排列交叉算子，保证子代仍然是合法排列（每个城市恰好一次）。

    ❌ 为什么不能直接切+粘？
    Parent A: 3 5 7 | 2 1 6 4 8
    Parent B: 8 2 1 | 5 3 7 6 4
    直接拼接 → Child: 3 5 7 5 3 7 6 4 ← 城市 5 和 3 重复了，城市 1、2、8 丢了。非法！

    最常用的 TSP 交叉算子 — OX (Order Crossover, 顺序交叉)

    OX 的想法很直观：从 Parent A 保留一段路径，其余空位按 Parent B 的访问顺序依次填，跳过已填过的城市。
    这样做的好处：子代既有 A 的部分路径结构，又继承了 B 的城市相对顺序。

    ③ 变异 — 排列编码的变异操作

    排列编码的变异必须保证仍然是合法排列。常用四种操作：

    操作做什么BeforeAfter
    Swap 随机选两个位置，交换 3 5 7 2 1 6 4 8 3 4 7 2 1 6 5 8
    Insertion 随机选一个城市，拔出，插到另一位置 3 5 7 2 1 6 4 8 3 7 2 1 5 6 4 8
    Inversion 随机选一段，反转（= 一次 2-opt） 3 5 | 7 2 1 | 6 4 8 3 5 | 1 2 7 | 6 4 8
    Scramble 随机选一段，内部随机打乱 3 5 | 7 2 1 | 6 4 8 3 5 | 2 7 1 | 6 4 8
    💡 Inversion 变异 = 随机 2-opt
    TSP 的 Inversion 变异本质上就是做一次随机的 2-opt。这不仅保持排列合法性，
    还对路径长度有实际影响——可能消除交叉。是 TSP-GA 中最推荐的变异操作。

    6.2 MA — Memetic 算法
    Memetic Algorithm (GA + Local Search)

    = 遗传算法 + 局部搜索

    在 GA 的每个阶段（生成初始种群后、交叉后、变异后）都嵌入局部搜索进行精细化打磨：

    Algorithm: Memetic Algorithm
1. 初始化种群 sp
2. 对 sp 中每个个体执行局部搜索        ← 关键差别
3. while 终止条件未满足:
4.    生成 spr ← 交叉 sp
5.    对 spr 执行局部搜索               ← 关键差别
6.    生成 spm ← 变异 spr 和 sp
7.    对 spm 执行局部搜索               ← 关键差别
8.    从 sp、spr、spm 中选择新种群 sp
9. return 种群中最优个体
    🧬 MA vs GA
    MA 将全局探索（GA 的种群多样性）+ 局部精化（LS 的快速收敛）结合， 在 VRP 类问题上通常显著优于纯 GA。代价是每次评估更昂贵。

    7. 核心设计原则

    7.1 跳出局部最优的四种策略

    策略机制代表算法
    1. 允许接受差解 以一定概率接受目标值更差的邻居，概率随「温度」降低而减小 SA
    2. 扰动当前解 对当前解施加较大幅度的扰动（shaking/perturbation），从新起点重新局部搜索 ILS
    3. 切换邻域结构 不同邻域中的邻居集合不同，在一个邻域中的局部最优可能在另一个邻域中仍有改进空间 VNS / VND
    4. 惩罚当前局部最优 修改目标函数，为局部最优解中的"特征"加惩罚项，迫使其离开当前区域 Guided Local Search

    7.2 探索 vs 利用 (Diversification vs Intensification)

    Intensification 利用Diversification 探索
    目标在已知优质区域深度搜索探索未充分搜索的区域
    做法小邻域局部搜索、精英保留随机扰动、大邻域、多种群
    风险陷入局部最优、早熟随机搜索、浪费算力
    🔑 没有银弹
    没有一种元启发式框架在所有问题上都优于其他。成功的应用源自： 仔细分析问题结构 + 精心设计搜索空间和算子。许多现代算法是多种方法的混合体（如 VNS + Tabu + SA）。

    7.3 加速技巧与实战经验

    技巧说明
    静态邻域限制VRP 中只考虑地理相近节点的 move
    增量评估\Delta = f(s_{new}) - f(s_{old}) 用 O(1) 计算，而非重新评估完整解

    下界剪枝计算 move 的下界，提前跳过不可能改进的 move
